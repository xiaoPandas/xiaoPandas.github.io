Java基础巩固一

发表于 2017-12-13 | 分类于 Java基础 |

在跟着视频学习的过程中，做一些笔记记录，以便之后进行复习和再一次巩固：

运行环境与编译器

  • 运行环境版本低于编译器版本时，会出现Bad version number in .class file错误；
  • 高版本的jdk可以向下兼容；
  • properties–Java Build Path 运行环境配置
    properties–Java Compiler 编译器配置

断点调试

  • f5：step into；单步执行时遇到子方法就跳入进去，然后继续单步执行
    f6：step over；单步执行时将子方法整个看作一步来执行
    f7：step return；单步执行到子方法内部时，一次执行完余下部分，并返回到上一层
  • drop to - frame：跳到当前方法的第一行；
    resume：跳到下一个断点；当没有下一个断点了的时候，结束调试；
    Breakpoints 视图：断点列表；
  • 注意：debug 结束之后要清空所有断点，然后关闭jvm！

常用快捷键

  • 内容提示：Alt + /
    快速修复：Ctrl + 1
    导包：ctrl + shift + O
  • 查看继承关系：ctrl + T
    格式化代码块：ctrl + shift +F
    添加注释：ctrl + shift + / - (要先选中需注释的代码)
  • 除去注释：ctrl + shift + (同样先选中)
    更改为大写：ctrl + shift + X
    更改为小写：ctrl + shift + Y
  • 复制行：ctrl + alt + 向下键
    查看源代码：ctrl + shift + T(或：ctrl + 鼠标)
    列出所有快捷键：ctrl + shift + L
  • 上下行交换内容或将当前行向上下移动：alt + 向上键/向下键
    前/后一个编辑的页面：alt + 向左键/向右键 (查看源代码时很有用)

JUnit 的使用

  • 对某个类的方法进行测试，建个专门的test类，里面有test方法；
    test方法的前面加上 @Test；
    对所有方法同时测试时右击类名进行测试，对单个方法进行测试时右击对应方法；
  • before()、after()方法：测试类中每一个测试方法运行的时候都运行；
    beforeclass()、afterclass()方法：一个测试类运行的时候只运行一次；

可变参数

允许方法定义长度可变的参数，可变参数只能处于参数列表的最后；
例 public void sum(int ...num) { } 调用方法时，编译器自动创建一个数组保存传递给方法的可变参数num，因此把可变参数num当作一个数组使用即可；

枚举

  • 何时使用：一些程序在运行的时候，它需要的数据不能是任意的，而必须是一定范围的值；
  • 如何使用：enum 关键字定义一个枚举类，可以有构造函数，字段和方法；（构造函数必须私有）
  • 注意事项：
    带抽象方法的枚举类在新建对象的时候要实现抽象方法；
    枚举是一个特殊形式的Java类，枚举中声明的每一个枚举值代表枚举类的一个实例对象；
    枚举可以实现接口，或继承抽象类；
    swich 语句可以接收一个枚举类型（jdk5扩展）；
    若枚举中只有一个枚举值，则可以当作单态设计模式使用（保证类对象在内存里面只有一个）；

内省

  • 内省访问 JavaBean 属性的两种方式：
    (1) 通过 PropertyDescriptor 类操作 Bean 的属性
    (2) 通过 Introspector 类获得 Bean 对象的 BeanInfo，然后通过 BeanInfo 来获取属性的描述器( PropertyDescriptor )，通过这个属性描述器就可以获取某个属性对应的 getter/setter 方法，然后通过反射机制来调用这些方法。
  • JavaBean 的字段有相应的 getter 或 setter 方法时可以称之为属性，有几个属性由有几个 getter 或 setter 方法决定而不由定义了几个决定。Object 类中还有一个 getClass() 方法，所以 JavaBean 都至少含有 class 属性。

泛型

  • 使用泛型时，泛型类型须为引用类型，不能是基本数据类型。
    如果两边都使用泛型，两边必须一样。
    List list = new ArrayList(); //可以
    List list = new ArrayList(); //可以
  • 编译器编译完之后会将class文件中的泛型擦除，以提高程序运行效率。
  • 泛型的基本术语：
    • ArrayList 中的 E 称为类型参数变量
    • ArrayList 中的 Integer 称为实际类型参数
    • 整个 ArrayList 称为泛型类型
    • ArrayList 称为参数化的类型(ParameterizedType)
