  3. 管道信息以 __ 进行写入和读出。
    字符流
    解析：
    管道是指用于连接一个读进程和一个写进程以实现进程之间通信的一种共享文件。向管道提供输入的是发送进程，也称为 写进程，数据的格式是字符流。接受管道数据的接受进程为读进程。

    缓冲池
    解析：

  6. 进程通信中， __ 常通过变量和数组形式实现。
    低级通信
    解析：
    进程通信分为低级通信和高级通信：
    低级通信：只能传递状态和整数值，传递信息量少，效率低，每次通信传递的信息量固定。
    高级通信：提高信号通信的效率，传递大量数据，减轻编程复杂度；分为三种方式：共享内存模式，消息传递模式，共享文件模式。

  13. 段页式管理每取一数据需要访问 __ 次内存。
    3
    解析：
    在段页式存储管理方式中，取一次指令或数据时，首先要从内存中查找段表；再查找该段对应的页表；根据页表拼接成物理地址后访问内存获得指令或数据。一共要进行 3 次内存访问。

  14. 操作系统中有一组常称为特殊系统调用，它不能被系统中断，在操作系统中称为 ___。

    解析：

    原语在管态下执行。

  15. MySQL 的默认 innodb 引擎事务隔离级别是 __ 。
    可重复读 (Repeatable read)
    解析：
    四类隔离级别，分别是：
    未提交读 (Read uncommitted)：一个事务可以看到其他未提交事务的执行结果；
    已提交读 (Read committed)：大多数数据库的默认事务隔离级别 (但 MySQL 不是)，一个事务只能看到已提交事务的结果；
    可重复读 (Repeatable read)：MySQL 的默认事务隔离级别，确保同一事务的多个实例在并发读取数据时，会看到同样的结果。
    可串行化 (SERIALIZABLE)：完全串行化的读，在每个读的数据行上都加上共享锁，大大降低了数据库的吞吐性能。

  16. 可以单独与 finally 语句使用的块是 __ 。
